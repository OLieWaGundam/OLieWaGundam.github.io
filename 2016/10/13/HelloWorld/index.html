HelloWorld

发表于 2016-10-13 | 分类于 随笔

契机

其实我作为一名小小的开发者，想拥有一个自己的博客很久啦，一年前舍友告知腾讯云有个“云+校园计划”，认证在校大学生之后就能“1元1月=服务器+域名”。于是我也跟风购买了一个服务器，并有了自己的域名：imwxj.cn。
这期间也有学过一点h5，也很羡慕舍友有一个自己的博客（imWosz）,但是一直没有利用这些资源去搭一个自己的服务器，一方面是觉得自己技术不够，感觉没什么能够分享的东西；另一方面是觉得搭一个博客好像很麻烦，懒得去捣鼓。
上个星期带刚好因为有任务需要装了个node的环境，然后就想起来好像有个叫Hexo的博客框架就是基于node的，刚好当助班也没什么事干了，不如就搭一个基于Hexo的博客玩玩吧，也算是督促自己学习。

实践

Hexo + GitHub Pages

因为之前有了解过一下，而且git和node也装好了，所以上手很快，直接百度“hexo + github pages”，果然出来一大堆教程，就随手点了一篇照着搭了。
大概流程就是先去github创建github pages，然后本地装个hexo，再把ssh放上github，hexo通过md文件生成静态界面，然后git到github pages。

简单两行命令之后，hexo的HelloWorld就出来了，然而默认的主题让我很不满意，于是我上网随意挑了一下，一眼相中Next这个简洁的主题。
最后，上腾讯云让自己的域名解析到github pages仓库的页面，于是，通过imwxj.cn就能访问自己的网站啦。

把博客部署到腾讯云服务器

回到宿舍跟舍友分享一下这件事情，舍友却一盆冷水泼我脸上。
“咦，打开好慢哦，你看看我的wosz，多快啊，别用github pages啦，github的服务器是国外的，放你腾讯云的服务器上多好”
果断用电脑ping一下：

果然，腾讯云的服务器要快很多。
好吧，既然要搞，那就再优化一下，而且服务器是之前就已经搭好了lnmp的环境的，应该也不会很麻烦。
两个想法：

  1. 服务器也装一个hexo，然后生成的文件放到www的文件夹下面，更新博客的方法就是本地写好md传到服务器，再到服务器hexo generate；
  2. 本地方法不变，git的地址改为服务器，就是本地生成静态文件再传到服务器;

显然，第二种方法更方便一点。
有了想法，当然是先百度一下啦，这次搜索到的东西就没有”hexo+github pages”那么多了，不过也算找到了一篇:使用 Git Hook 自动部署 Hexo 到个人 VPS;
大概就是在服务器建一个git的裸仓库，然后通过git hooks把本地git上去的静态文件放到www的文件夹里。
虽然原理不是很难，但是具体实施起来对我来说还是有点难度的，因为对linux系统不够熟悉，期间也是踩了一点坑，包括新建git用户的权限问题，vim的使用，ssh不生效等，不过最后还算是完成啦，虽然每次git上去都要输密码，不过目前还没找到解决办法，这也不是什么大问题，就不继续捣鼓了。

搭完后的想法

感觉总算是有了自己的一片小天地，甚至有种小时候刚开始玩QQ空间的感觉哈哈。
总之，希望这能激励自己好好学点东西，然后分享上来吧。等以后回头看看自己写的东西，不管写得好还是不好，那感觉应该很棒吧。
At Last, Hello World, Gundam‘s Base。
